1 · Resolved Open Questions from 4D

All five design decisions left open after the first draft are now resolved.

✓

Cache invalidation granularity

Modifying a node's configuration invalidates only that node's cache. Upstream nodes stay green. This means "Run from Filter" still uses the cached Data Table output even if Filter itself was edited.

Decision: Configuration changes invalidate; connection re-routing invalidates the re-routed node and all downstream nodes.
✓

Filter presets scope

Per-workflow scope. Each workflow maintains its own preset list. Cross-workflow reuse is not needed for the target user profile (single-workflow builders); global presets add complexity with minimal benefit.

Decision: Per-workflow. If demand emerges, can be promoted to global later.
✓

Tables panel placement

Slide-out overlay wins over persistent sidebar panel and modal. Slide-out preserves canvas space for most editing sessions; appears only on demand; does not block editing while open.

Decision: Slide-out panel triggered by the 'Tables' toolbar button.
✓

Cost warning for LLM nodes

Implemented as V2c. Cost modal appears before execution when any node has a known per-call cost (OpenAI, Anthropic, etc.). The user can opt out per-workflow. This uses per-provider static pricing data embedded in n8n.

Decision: Show modal by default; add 'Don't show for free-only workflows' toggle.
✓

Mobile / responsive indicators

Deprioritized. The target users (workflow builders) work on desktop. Cache status indicators in the execution log are sufficient for occasional mobile checks. Not included in V2.

Decision: Not in scope for this redesign cycle.

2 · Final Design Components (V2)

Six refined components from V1 and three new components first introduced in V2. CSS wireframes are the primary deliverable; AI-generated image placeholders supplement the two most spatially complex new designs.

Image vs. wireframe rationale: CSS wireframes (like those in 4C and 4D) are used for all components — they are consistent, dark-mode-aware, and always current. AI-generated Balsamiq-style images are provided as supplemental placeholders for V2c and V2d only, where the spatial layering (modal over canvas, popover over panel) is harder to communicate in inline CSS. The generation prompts are embedded below each placeholder.

Motivation: The 4D open question asked whether “Run from here” should estimate API cost before executing. The answer is yes — but the estimate belongs at the global execution level (the Execute button), not per-node. The modal intercepts only workflows with at least one paid node.

Design decisions: Per-node cost table (not just a total) helps users identify which node drives cost. The “Don’t show for free-only workflows” checkbox respects expert users who build purely free workflows. This follows Nielsen H1 (Visibility of Status) applied to financial status — the system makes invisible costs visible before they are incurred.

Pricing data source: n8n already detects which AI nodes are configured; static pricing tables per provider (OpenAI, Anthropic, etc.) can be bundled with the app and updated via node package versions.

Input Mode Switch Confirmation

NEW in V2
New in V2: Addresses H3 violation identified in 4B — switching “Input Data Mode” silently discards entered field values.
V2d — Input Mode switch: confirmation before silent data loss (NEW)
HTTP Request — Configure

NEW in V2 ⚠ Switching to Expression will clear “12345”

Your fixed value will be lost. You can restore it manually. Continue?

Motivation: A user types a Profile ID (“12345”) into a Fixed Value field, then accidentally clicks Expression mode — the value silently disappears. There is no undo. This was observed with P1 during the 4C user study (Task 3) and is a textbook Nielsen H3 (User Control and Freedom) violation.

Design decisions: A simple inline popover (not a full modal) keeps the confirmation close to the action. Two asymmetric buttons — “Keep value, cancel” (safe, default-styled) and “Switch (clear value)” (destructive, filled) — make the safer choice easier to click. This follows Norman’s principle of forcing functions: the system forces a deliberate choice before the irreversible action.

Tradeoff: The confirmation adds one click for users who intentionally switch modes. Mitigated by making “Switch” the smaller button so it is slightly harder to click accidentally but still accessible.

Motivation: The heuristic analysis (4B, H8) flagged that the Data Table node lists 11 actions in a flat, undifferentiated list. New users spend significant time scanning it. The natural split — 7 Row Actions + 4 Table Actions — already exists conceptually in n8n’s documentation.

What changed from V1: V1d added a reset button and presets. V2f adds the AND/OR toggle. n8n’s current Filter node supports AND/OR logic but exposes it deep in the node settings — it is not visible on the main conditions list. Promoting it to the top of the conditions panel makes the logic explicit and adjustable without digging.

Open question #2 resolved: Presets are per-workflow. The decision is based on n8n’s primary usage pattern — each workflow has a distinct domain and data schema, so a preset built around one table’s columns is unlikely to be reusable in another workflow. Per-workflow scope keeps presets relevant and avoids a long shared list.

4 · Design Rationale — Principles Behind the Evolution

How each core design principle shaped decisions across all three versions.

Nielsen H6 — Recognition Over Recall

V1a moved Data Tables to the sidebar (visible). V2a adds a first-use tooltip and a toolbar shortcut in the editor. Both surfaces now surface the feature without requiring the user to remember where it is.

Nielsen H1 — Visibility of System Status

V1b introduced per-node cache dots. V2a refines the labels (cached / modified / not run). V2c adds the cost modal which makes API cost visible before it is incurred — a form of 'financial status visibility.'

Nielsen H5 — Error Prevention

V1c wizard prevents bad imports. V2b's Step 2 preview prevents committing a wrong dataset. V2d's confirmation popover prevents accidental field value loss. V1f's auto-naming prevents table confusion in dropdowns.

Nielsen H3 — User Control and Freedom

V1d reset button restores control in filters. V2d's 'Keep value, cancel' button is the emergency exit for mode switching. V2c's 'Cancel' stops costly execution before it starts.

Nielsen H7 — Flexibility and Efficiency of Use

V1b's 'Run from here' lets experts skip unchanged upstream nodes. V2f's AND/OR toggle and presets accelerate complex filter authoring. V2e's action search lets power users jump directly to the action they need.

Norman — Affordance and Constraints

V2e groups node actions with labeled categories, so the visual structure constrains choice and guides scanning. V2c's per-node cost table makes the cost source transparent, not just the total.
