Phase 3B — User Study

Artifacts & Links

Canvas Forum Post

Overview

In Phase 3B, each team member conducted an individual user study observing real users interacting with Google Looker Studio. The goal was to understand what users do, what they want to do, and how the interface helps or hinders them — especially when things go wrong. All findings were posted to the Canvas forum and synthesized here.

Study Method

Each team member conducted a single-user observational session with think-aloud protocol. The studies were naturalistic where possible, with participants performing realistic report-building workflows. Users were asked to verbalize their reasoning while completing assigned tasks.

Team Member Studies

Ke Ding's User Study

Participant: User A — Graduate student in Mechanical Engineering with basic experience in Excel/MATLAB visualization but no prior Looker Studio experience.

Tasks Given:

  1. "Please look at the Ecommerce Metrics section on the dashboard and connect the CSV file I provided to this report and make it the data source for the dashboard."
  2. "Please modify one of the chart types in the dashboard and make it use the data from the uploaded CSV file instead of the original sample dataset."

Key Findings:

  • Data source confusion: User successfully uploaded the CSV file but did not realize the chart was still using the original sample data. They assumed the dashboard automatically switched to the new dataset because it appeared in the Data panel.
  • Chart type vs. style confusion: When asked to change the chart type, the user modified the chart style (line to bar) rather than using the collapsed Chart Type option. The option was not immediately visible.
  • Signup flow issue: The "Continue" button appeared grayed out on the preferences page, causing confusion about whether selections were required.

Redesign Speculations: Add clearer feedback after dataset upload explaining that charts still need to be manually connected. Make the chart type option expanded by default.

Ziyi Zhang's User Study

Participant: User B — Graduate student in UIUC iSchool with data visualization experience in R and Python but new to Looker Studio's report-editing conventions.

Task Given: "Please use Google Looker Studio to import the file corgs_per_country_over_time_columns_2020.csv, create any visualization(s) you think are appropriate, make the visualizations interactive, and organize them into a report that includes annotations and interactive visualizations. Please think aloud during the process."

Key Findings:

  1. Report renaming slip: When clicking the title, the system did not auto-select the full default title, causing new text to be inserted mid-title. User expected behavior similar to Google Docs.

  2. Chart field selection confusion: The properties panel overloaded a single area with multiple meanings (field identity, type, edit behavior). Different clickable zones performed different actions without clear differentiation.

  3. Sorting discovery failure: User could not find the sort control independently. They tried right-clicking the chart (no feedback) and clicking various field areas before needing guidance.

  4. Layout constraint confusion: After switching chart types, axis labels were partially hidden. User could not understand whether the issue came from chart type, canvas size, style settings, or responsive layout behavior.

  5. System freeze: During exploratory style changes, the system froze completely, requiring session restart.

  6. Text annotation frustration: In responsive layout, text boxes snapped to unexpected positions. Font size changes caused selection loss and unexpected text replacement. User abandoned responsive mode for freeform.

  7. Precise targeting required: Even in freeform mode, moving text boxes required targeting the border precisely; clicks often entered text-edit mode instead.

Relation to Nielsen's Heuristics:

  • Visibility of system status: Poor feedback about why labels were cut off
  • Match between system and real world: Field controls didn't match office-software expectations
  • User control and freedom: Responsive snapping reduced feeling of control
  • Consistency and standards: Violated PowerPoint/Google Docs expectations
  • Error prevention: Many preventable slips (mid-title insertion, ambiguous click targets)
  • Recognition rather than recall: Users had to remember where sorting lived

Miantong Zhang's User Study

Participant: User C — Graduate student who frequently uses Google Docs and Excel but has no previous Looker Studio or data visualization platform experience.

Tasks Given:

  1. Create a simple report
  2. Connect a dataset
  3. Add a chart to the dashboard
  4. Export or share the report

Key Findings:

  • Upload button not found: User expected a clear "Upload File" button when adding data. The data connector system was not intuitive.
  • CSV connector uncertainty: User hesitated before selecting the CSV connector, unsure if it was the correct option for uploading a local dataset.
  • Add Chart hidden: User initially looked for visualization tools in the dataset panel, not the top toolbar. Took significant time to locate the Add Chart button.
  • Export was easy: The File > Download option was clearly visible and straightforward.

Redesign Speculations: Add a visible "Upload Dataset" button. Place Add Chart function closer to the dataset panel or make it more prominent.

Mianchen Zhang's User Study

Participant: User D — Graduate student from UIUC with moderate programming background (R, Python) and strong database experience (MySQL).

Task Given: "Please try to find a template in Google Looker Studio that you think could be used to analyze country-level data."

Key Findings:

  • Template Gallery not visible: User created a blank report first instead of finding templates. The colorful "+ Blank Report" button was more visually prominent than the Template option.
  • Searched for templates inside editor: User clicked multiple ellipsis menus trying to find a template option, expecting templates could be inserted like components.
  • Long template browsing time: User spent approximately 8 minutes browsing template categories.
  • Repair icons caused confusion: After applying a template designed for different data, visualization panels showed repair icons without clear explanation of what was wrong.

Redesign Speculations: Make Template Gallery more visible. Display recommended templates when creating a new report. Provide clearer feedback when templates don't match the uploaded dataset structure.
