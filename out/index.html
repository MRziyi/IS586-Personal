A solo usability study targeting the workflow editor and Data Tables feature — data setup, CSV import, node testing, and execution flow for AI-assisted pipelines. Audience: novice-to-intermediate n8n users.

Top Issues Found (4B)

High Medium

No execution caching

Every edit forces a full re-run — including expensive LLM/API calls — with no cost warning.

Duplicate table names on copy

Duplicated Data Tables keep the same name, making them indistinguishable in dropdowns across the workflow.

Stale pinned data silently persists

Unpinning test data reverts silently; users can work with stale data for long stretches without realising it.

Data Tables buried in home screen

Not findable in the workflow editor — users must navigate back to the home screen to manage tables.

Silent CSV import failures

Type mismatches during import produce no warning; columns silently fail or receive the wrong type.

Destructive mode switch

Switching "Input Data Mode" discards existing field content without any confirmation prompt.
